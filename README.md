## Тестовое задание: форма управления учетными записями

Стек: Vue 3 + Composition API + TypeScript + Pinia + Element Plus.

### Что требовалось

- Реализовать форму управления учетными записями со списком и кнопкой добавления.
- Поля записи:
  - Метка (необязательное), max 50, ввод через `;`, при сохранении преобразуется в `[{ text: '...' }]`.
  - Тип записи (обязательное): `LDAP` или `Локальная`.
  - Логин (обязательное), max 100.
  - Пароль (обязательное только для `Локальная`), max 100; для `LDAP` скрыт и сохраняется как `null`.
- Логика:
  - `+` добавляет пустую запись в конец списка.
  - Удаление полностью удаляет запись.
  - Валидация по окончанию ввода: `blur` для текстовых, `change` для select.
  - При валидности запись сохраняется/обновляется в стейт-менеджере; иначе поля подсвечиваются ошибкой.
- Данные должны сохраняться в Pinia и отображаться после перезагрузки страницы.

### Что сделано

- Реализован UI формы: заголовок, кнопка `+`, подсказка для “Метки”, список учетных записей.
- Добавление/удаление записей, условный показ поля “Пароль” для `Локальная`.
- Валидация по правилам задания и подсветка ошибок через компоненты Element Plus.
- Преобразование меток `dev;admin` -> `[{ text: 'dev' }, { text: 'admin' }]`.
- Pinia store с персистентностью в `localStorage` и восстановлением при загрузке.

### Идеи улучшений

- Улучшить UX валидации: показывать ошибки только для “тронутых” (touched/dirty) полей, а не сразу для всей записи.
- При переключении типа на `LDAP` очищать `draft.password`, чтобы скрытое поле не держало значение в состоянии.
- Оптимизировать бандл: подключать Element Plus “по компонентам”/через резолвер и/или добавить code-splitting.
- Добавить минимальные тесты на доменную логику (`parseLabels`, `validateAccountDraft`, `draftToAccount`) и базовые скрипты `lint/format`.
